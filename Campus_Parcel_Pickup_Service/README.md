项目概述
校园快递代取项目旨在为佛山大学学生搭建一个互助式快递代取平台。需要代取的学生可发布任务，有空闲的学生可接单并实时更新进度（接单、途中、已完成）。平台提供三方端（客户端、跑腿端、客服端），支持身份认证、隐私保护、订单追溯与客服协调，解决校园快递点距离远、取件时间冲突等问题。

核心模块

1. 用户认证与权限管理模块
   学生使用学号、姓名、手机号完成实名注册，提交个人身份信息（学院、宿舍区等）。

系统区分客户端与跑腿端身份，跑腿端需额外审核信用资质。

客服端拥有最高数据查看与管理权限。

2. 任务发布与接单模块（客户端/跑腿端）

客户端：填写快递信息（取件码、快递点、重量、备注），发布代取任务，设置悬赏金额。

跑腿端：查看待接任务列表，一键接单，接单后任务状态变更为“接单中”。

系统自动匹配同宿舍区任务，优先推荐给附近跑腿员。

3. 任务进度跟踪与状态管理模块（跑腿端主导）
   
跑腿员可修改任务状态：接单 → 途中（已取到快递） → 待确认（已送达客户指定地点）。
客户端收到“待确认”推送后，需在24小时内点击“确认收货”，任务状态变为“已完成”。若客户未确认且未申诉，系统自动确认。
客户可在“待确认”状态下发起申诉（如未收到件、物品损坏），客服端介入处理。

4. 隐私保护与订单信息共享模块（两端后台）

跑腿端接单后，仅显示客户昵称、快递点、宿舍楼号，隐藏手机号全号与具体房号。

双方可查看当前订单详情（取件码、时间线）及历史订单记录。

订单完成后7天自动脱敏归档，保护用户隐私。

5. 客服协调与数据管理模块（客服端）

客服端可查询所有客户、跑腿员、订单的完整信息（含敏感数据）。

内置即时通讯窗口，支持客服与客户/跑腿员双向私聊，用于纠纷调解、异常协调。

提供数据统计（订单量、完成率、投诉率），辅助运营决策。

技术栈

简化实现：
- 前端：原生 HTML + CSS + JavaScript
- 后端：PHP（简单 MVC，处理任务发布/接单/状态流转）
- 数据库：MySQL（存储用户、订单、消息记录）
- 部署：Nginx + PHP-FPM

完全实现：
后端
核心框架：Spring Boot 2.x / 3.x


微服务（可选）：Spring Cloud（若拆分为用户、订单、客服等独立服务）

持久层：MyBatis-Plus / JPA（Hibernate）

接口协议：RESTful API + JSON

实时通信：WebSocket（用于状态推送、客服聊天）

安全框架：Spring Security + JWT（令牌认证）

前端（Web 管理端/客服端）
框架：Vue 3 / React 18

UI 库：Element Plus / Ant Design

状态管理：Pinia / Redux

移动端（客户端/跑腿端）
原生：Android 使用 Java（与后端统一语言）

跨平台：Flutter 或 Uni-app（开发效率高，一套代码双端）

网络请求：OkHttp / Retrofit

数据库
关系型：MySQL 8.0（存储用户、订单、历史记录）

缓存：Redis（会话、热点数据、分布式锁）

文档型（可选）：MongoDB（存储聊天记录等非结构化数据）

部署与运维
容器化：Docker + Docker Compose

服务器：Nginx（反向代理）

版本控制：Git

第三方服务
消息推送：极光推送 / 个推（订单状态通知）

短信验证：阿里云/腾讯云短信

对象存储：阿里云OSS / 七牛云（存放用户头像、凭证图片）
